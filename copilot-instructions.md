# Regras para mensagens de commit

- Primeira linha: resumo curto (máx. 50 caracteres).
- Segunda linha: em branco.
- Linhas seguintes: explicação detalhada das mudanças.
- Sempre usar prefixos: feat, fix, docs, refactor, test, chore.
- Explicar o motivo da mudança, não só o que foi alterado.
- Evitar frases vagas como "alterações diversas".
- **Todas as mensagens de commit devem ser escritas em inglês.**
